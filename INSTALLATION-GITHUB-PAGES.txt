Own Light Website - GitHub Pages Ready fixed v10
====================================

This package is STATIC ONLY. It does not contain PHP, .htaccess, or server-side proxy files because GitHub Pages cannot run PHP.

WHAT WAS FIXED
--------------
- GitHub Pages hosting issue caused by the old PHP-based website package.
- Website now runs as static HTML/CSS/JS on GitHub Pages.
- Website connects directly to the content API using browser fetch.
- Saved updates appear on the website after you publish changes.
- Multiple product photos/gallery are supported.
- Product size, colour, discount label, discount %, compare price, and category discounts are supported.
- Category product filtering, product detail links, WhatsApp inquiry, Instagram link, and inquiry saving are supported.
- No bundled product/category images are included. Upload all images from the editor.
- .nojekyll and a GitHub Pages workflow are included.
- Product video/social/location links from CMS show under product photos and on product cards.
- Anonymous product view, inquiry click, WhatsApp click, and inquiry-open tracking is sent to the Netlify CMS API.

SETUP STEPS
-----------
1. Deploy the content backend package to Netlify first.
2. Open this URL to confirm the API works:
   https://your-site.netlify.app/api/health
3. Edit config.js in this website package.
4. Replace:
   https://your-site.netlify.app/api
   with your real Netlify API URL.
5. Upload/push all website files to your GitHub repository root.
6. In GitHub repository settings, open Pages.
7. Use GitHub Actions as the Pages source, or deploy from branch if you prefer.
8. Re-run the failed Pages deployment.

IMPORTANT
---------
Do not put admin tokens or GitHub tokens in config.js. The public website only needs the public API URL.

If the GitHub Pages deploy screen still fails, click the red deploy job and copy the exact error line. The package itself is static and ready for GitHub Pages.
